Build a command-line todo app named todo.py in the current directory that persists state between runs in a file todo.json (in the current directory). It must support exactly these commands:
- `python todo.py add "TEXT"`  -> append a new task with the given text
- `python todo.py done N`      -> mark task number N (1-based) as done
- `python todo.py list`        -> print every task, one per line, in insertion order, formatted exactly as: `N. [ ] TEXT` for open tasks and `N. [x] TEXT` for done tasks (N is 1-based).
State MUST survive across separate process invocations (persist to todo.json). After implementing, test the whole flow yourself by running several commands.
